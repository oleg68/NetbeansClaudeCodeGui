# Ручное тестирование MCP-интеграции (Stage 10)

## Подготовка

1. Собрать плагин: `mvn package -DskipTests`
2. Установить NBM: Tools → Plugins → Downloaded → Add Plugins → выбрать `target/nb-claude-code-gui-0.10.0-SNAPSHOT.nbm`
3. Перезапустить NetBeans
4. Открыть любой Java-проект (нужен для тестов с файлами и проектами)

---

## TC-01: MCP-сервер запускается при старте IDE

**Цель:** убедиться, что WebSocket-сервер стартует и создаёт lock-файл.

**Шаги:**
1. Запустить NetBeans с установленным плагином
2. Подождать ~3 секунды после полной загрузки IDE

**Проверки:**
- `ls ~/.claude/ide/` — должен появиться файл вида `8990.lock` (или другой порт из диапазона 8990–9100)
- `cat ~/.claude/ide/*.lock` — должен содержать JSON вида:
  ```json
  {"pid":12345,"ideName":"NetBeans","transport":"ws","authToken":"...","workspaceFolders":["/path/to/project"]}
  ```
- В `~/.netbeans/28/var/log/messages.log` должна быть строка:
  `Claude Code MCP server started on port XXXX`

## TC-02: Lock-файл обновляется при открытии/закрытии проекта

**Цель:** убедиться, что `workspaceFolders` в lock-файле актуален.

**Шаги:**
1. Открыть второй проект: File → Open Project
2. Проверить `cat ~/.claude/ide/*.lock`
3. Закрыть один из проектов
4. Проверить снова

**Проверки:**
- После открытия второго проекта — в `workspaceFolders` два пути
- После закрытия — один путь

---

## TC-03: Claude сессия стартует с --mcp-config

**Цель:** убедиться, что Claude CLI получает конфиг нашего MCP-сервера.

**Шаги:**
1. Открыть сессию Claude: кнопка на тулбаре или правый клик на проекте → Open with Claude Code
2. Выбрать директорию проекта, нажать Open
3. Дождаться запуска Claude в терминале

**Проверки:**
- В терминале Claude написать `/mcp` и нажать Enter
- Должен отобразиться сервер `netbeans` в списке подключённых MCP-серверов со статусом connected
- В `messages.log`: строка `Claude PTY started` без ошибок

## TC-04: getWorkspaceFolders

**Цель:** Claude может получить список открытых проектов.

**Шаги:**
1. В сессии Claude написать:
   ```
   What projects do I have open? Use the getWorkspaceFolders tool.
   ```

**Проверки:**
- Claude вызывает инструмент `getWorkspaceFolders`
- В ответе — пути к открытым проектам, совпадающие с реально открытыми в IDE

1. Открыть 2–3 файла в редакторе NetBeans

- Claude вызывает `getOpenEditors`

## TC-06: getCurrentSelection

**Цель:** Claude читает выделенный текст.

**Шаги:**
1. Открыть любой Java-файл
2. Выделить несколько строк кода мышью
3. В сессии Claude написать:

- Возвращённый текст совпадает с выделением в редакторе

## TC-09: checkDocumentDirty

**Цель:** Claude определяет наличие несохранённых изменений.

**Шаги:**
1. Открыть файл и внести изменение (не сохранять)
2. В сессии Claude написать:
   ```
   Check if [путь к файлу] has unsaved changes.
   ```
3. Сохранить файл (Ctrl+S), спросить снова

**Проверки:**
- До сохранения: `isDirty: true`
- После сохранения: `isDirty: false`

## TC-10: saveDocument

**Цель:** Claude сохраняет документ.

**Шаги:**
1. Открыть файл и внести изменение (не сохранять — заголовок вкладки показывает `*`)

   ```

   ```

**Проверки:**

- Изменения записаны на диск

## TC-11: getDiagnostics

**Цель:** Claude получает ошибки компилятора.

**Шаги:**
1. Открыть Java-файл и внести синтаксическую ошибку (например, удалить `;`)
2. Подождать, пока NetBeans подчеркнёт ошибку
3. В сессии Claude написать:
   ```
   What are the current errors in [путь к файлу]?
   ```

**Проверки:**
- Claude вызывает `getDiagnostics`
- В ответе — описание ошибки, номер строки, severity `ERROR`

## TC-16: Несколько IDE одновременно

**Цель:** плагин не конфликтует с другой IDE (например, VSCode с MCP).

**Шаги:**
1. Запустить VSCode с Claude Code расширением (если есть)
2. Запустить NetBeans с нашим плагином
3. Открыть сессию Claude в NetBeans

**Проверки:**
- В `~/.claude/ide/` — два lock-файла на разных портах
- `/mcp` в Claude-сессии NetBeans показывает только `netbeans`-сервер
- Инструменты работают с NetBeans, не с VSCode

## Что делать при ошибках

| Симптом | Где смотреть |
|---------|-------------|
| Lock-файл не создаётся | `~/.netbeans/28/var/log/messages.log` — ошибки в `ClaudeCodeInstaller` |
| `/mcp` не показывает netbeans | Проверить temp-файл конфига: в `messages.log` должен быть путь `/tmp/nb-mcp-*.json`; проверить его содержимое |
| Инструмент возвращает ошибку | `messages.log` — исключения в `NetBeansMCPHandler` |
| Claude не видит выделение | Убедиться, что файл открыт через NetBeans editor, а не внешний |
